The Multiple Testing Problem

When you run a single statistical test at the 5% significance level, there is a 5% chance of a false positive. That sounds manageable. But what happens when you run many tests at once?

The curve above shows how quickly certainty collapses. At 20 tests, you have a 64% chance of at least one false positive. At 100 tests, it is nearly guaranteed.

Correction Methods

Bonferroni

The simplest fix: divide the significance threshold by the number of tests. If you run 100 tests, each test must clear p < 0.0005 instead of p < 0.05. This controls the family-wise error rate (FWER), the probability that any single test is a false positive. It is strict, and it works. But it can be too conservative, especially when many of the tests are testing real effects.

Benjamini-Hochberg (BH)

A more modern approach that controls the false discovery rate (FDR), the expected proportion of rejected hypotheses that are false positives. Instead of a fixed threshold, it uses a rising line.

Show the code 
np.random.seed(42)

n_tests = 100
n_true = 20

p_null = np.random.uniform(0, 1, n_tests - n_true)
p_true = np.random.beta(1, 20, n_true)

p_values = np.concatenate([p_true, p_null])
is_true = np.array([True]*n_true + [False]*(n_tests - n_true))

sorted_idx = np.argsort(p_values)
p_sorted = p_values[sorted_idx]
is_true_sorted = is_true[sorted_idx]

ranks = np.arange(1, n_tests + 1)
bh_threshold = ranks * 0.05 / n_tests

bh_cutoff = 0
for k in range(n_tests):
    if p_sorted[k] <= bh_threshold[k]:
        bh_cutoff = k + 1

fig, ax = plt.subplots(figsize=(10, 5))

colors = ['#2c3e50' if t else '#b8945a' for t in is_true_sorted]
for i in range(n_tests):
    marker = 'o' if is_true_sorted[i] else 's'
    ax.scatter(ranks[i], p_sorted[i], c=colors[i], s=30, marker=marker, zorder=3, alpha=0.8)

ax.plot(ranks, bh_threshold, color='#b8945a', linewidth=2, linestyle='--', label='BH threshold (q=0.05)')

if bh_cutoff > 0:
    ax.axvline(x=bh_cutoff, color='#c27c5a', linewidth=1.5, linestyle=':', alpha=0.7)
    ax.annotate(f'BH cutoff at rank {bh_cutoff}', xy=(bh_cutoff, p_sorted[bh_cutoff-1]),
                fontsize=10, color='#c27c5a', fontweight='bold',
                xytext=(bh_cutoff + 10, p_sorted[bh_cutoff-1] + 0.15),
                arrowprops=dict(arrowstyle='->', color='#c27c5a', lw=1.2))

ax.fill_between(ranks[:bh_cutoff], 0, p_sorted[:bh_cutoff], color='#2c3e50', alpha=0.08)

The BH procedure sorts all p-values from smallest to largest, then draws a line from (1, 0.05/m) to (m, 0.05). Any p-value that falls below this line is rejected. The key insight: it finds the largest rank k where p(k) is still below the threshold, then rejects everything from rank 1 through k.

When to Use Which

Use Bonferroni when:

  • The cost of a single false positive is catastrophic (clinical trials, genomics with functional follow-up)
  • You are running a small number of planned comparisons
  • You need to guarantee FWER control

Use Benjamini-Hochberg when:

  • You are screening many hypotheses and expect some real effects
  • You can tolerate a controlled proportion of false discoveries
  • Power matters more than absolute certainty (exploratory analysis, feature selection)

Key Takeaways

  1. Running multiple tests inflates false positives exponentially
  2. Bonferroni divides the threshold by the number of tests (strict, controls FWER)
  3. Benjamini-Hochberg uses a rising threshold line (flexible, controls FDR)
  4. Neither is universally better. The right choice depends on the cost of errors in your specific context

## The Multiple Testing Problem

When you run a single statistical test at the 5% significance level, there is a 5% chance of a false positive. That sounds manageable. But what happens when you run many tests at once?

## Correction Methods

### Bonferroni

The simplest fix: divide the significance threshold by the number of tests. If you run 100 tests, each test must clear p < 0.0005 instead of p < 0.05. This controls the **family-wise error rate (FWER)**, the probability that any single test is a false positive. It is strict, and it works. But it can be too conservative, especially when many of the tests are testing real effects.

### Benjamini-Hochberg (BH)

A more modern approach that controls the **false discovery rate (FDR)**, the expected proportion of rejected hypotheses that are false positives. Instead of a fixed threshold, it uses a rising line.

```{python}
#| label: fig-bh-procedure
#| fig-cap: "The Benjamini-Hochberg procedure: sorted p-values vs. the BH threshold line"
np.random.seed(42)

n_tests = 100
n_true = 20

p_null = np.random.uniform(0, 1, n_tests - n_true)
p_true = np.random.beta(1, 20, n_true)

p_values = np.concatenate([p_true, p_null])
is_true = np.array([True]*n_true + [False]*(n_tests - n_true))

sorted_idx = np.argsort(p_values)
p_sorted = p_values[sorted_idx]
is_true_sorted = is_true[sorted_idx]

ranks = np.arange(1, n_tests + 1)
bh_threshold = ranks * 0.05 / n_tests

bh_cutoff = 0
for k in range(n_tests):
    if p_sorted[k] <= bh_threshold[k]:
        bh_cutoff = k + 1

fig, ax = plt.subplots(figsize=(10, 5))

colors = ['#2c3e50' if t else '#b8945a' for t in is_true_sorted]
for i in range(n_tests):
    marker = 'o' if is_true_sorted[i] else 's'
    ax.scatter(ranks[i], p_sorted[i], c=colors[i], s=30, marker=marker, zorder=3, alpha=0.8)

ax.plot(ranks, bh_threshold, color='#b8945a', linewidth=2, linestyle='--', label='BH threshold (q=0.05)')

if bh_cutoff > 0:
    ax.axvline(x=bh_cutoff, color='#c27c5a', linewidth=1.5, linestyle=':', alpha=0.7)
    ax.annotate(f'BH cutoff at rank {bh_cutoff}', xy=(bh_cutoff, p_sorted[bh_cutoff-1]),
                fontsize=10, color='#c27c5a', fontweight='bold',
                xytext=(bh_cutoff + 10, p_sorted[bh_cutoff-1] + 0.15),
                arrowprops=dict(arrowstyle='->', color='#c27c5a', lw=1.2))

ax.fill_between(ranks[:bh_cutoff], 0, p_sorted[:bh_cutoff], color='#2c3e50', alpha=0.08)

ax.scatter([], [], c='#2c3e50', s=30, marker='o', label='True effect')
ax.scatter([], [], c='#b8945a', s=30, marker='s', label='Null (no effect)')

ax.set_xlabel('Rank (sorted p-values)', fontsize=12)
ax.set_ylabel('p-value', fontsize=12)
ax.set_title('Benjamini-Hochberg Procedure', fontsize=14, color='#2c3e50', fontweight='bold')
ax.legend(fontsize=10, framealpha=0.9)
ax.set_xlim(0, 105)
ax.set_ylim(-0.02, 1.05)
ax.grid(True, alpha=0.1, color='#2c3e50')
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)

fig.tight_layout()
plt.show()
```

The BH procedure sorts all p-values from smallest to largest, then draws a line from (1, 0.05/m) to (m, 0.05). Any p-value that falls below this line is rejected. The key insight: it finds the largest rank k where p(k) is still below the threshold, then rejects everything from rank 1 through k.

## When to Use Which

**Use Bonferroni when:**

- The cost of a single false positive is catastrophic (clinical trials, genomics with functional follow-up)
- You are running a small number of planned comparisons
- You need to guarantee FWER control

**Use Benjamini-Hochberg when:**

- You are screening many hypotheses and expect some real effects
- You can tolerate a controlled proportion of false discoveries
- Power matters more than absolute certainty (exploratory analysis, feature selection)

## Key Takeaways

1. Running multiple tests inflates false positives exponentially
2. Bonferroni divides the threshold by the number of tests (strict, controls FWER)
3. Benjamini-Hochberg uses a rising threshold line (flexible, controls FDR)
4. Neither is universally better. The right choice depends on the cost of errors in your specific context
