The five drawers above are how I’d shelve it. This is what I’d say to a friend over coffee about each one.

claude.ai is the front door, and most people stop there

Most people meet Claude through claude.ai and never go further. That’s fine. But the chat box on its own is maybe twenty percent of what’s possible.

The real unlock is Projects. A persistent workspace with files and instructions. Once you stop starting from zero every conversation, your time-to-useful-output drops by half. Pair it with Memory (which learns your preferences over time) and Preferences (which lets you tell Claude things explicitly), and you have an assistant that actually remembers you.

Artifacts are the underrated star. A working document, an interactive chart, a mini-app, a calculator, all generated in conversation and editable live. Stop thinking of Claude as a chatbot. Start thinking of it as a workshop.

Skills in claude.ai are packaged playbooks for specific jobs (a polished pptx, a financial model, a press release in your voice). Connectors plug Claude into Gmail, Calendar, Drive, Slack, and more. Together, they turn “ask Claude” into “have Claude do it”.

If you only learn one thing from this whole guide, learn how to use Projects well. The compounding return is huge.

Claude Code is the place I do my real work

Claude Code is the agentic terminal. The seven building blocks (CLAUDE.md, Skills, MCP, Hooks, Subagents, Plugins, Agent Teams) layer like a stack. Most days you only need two or three.

I always start with CLAUDE.md at the project root. Standing rules, conventions, the things I’d otherwise repeat every morning. Then I add Skills for the workflows I keep doing (creating a Quarto post, cleaning a CSV, generating a chart in my brand palette). MCP comes in when I need real data Claude can’t see.

Hooks are the underrated layer. A shell command that runs at a specific point in Claude’s lifecycle. I use one to grep my rendered HTML after every push, because Claude Code sometimes claims success without actually writing the file.

Subagents protect the main session when work gets clogged. Plugins are how good setups travel between repos and teammates. Agent Teams are the parallel mode, experimental and token-heavy, worth using only when parallelism actually pays off.

The API is the structural layer everything else sits on

If claude.ai is the front door and Claude Code is the workshop, the API is the foundation. The same Claude, exposed as a single endpoint (/v1/messages) that you POST to.

Five things to know:

  • Messages is the single endpoint that does almost everything. Tool use, vision, thinking, batch, all happen through this one endpoint with extra fields.
  • Tool use is how Claude calls your code. Declare a function, hand it over, watch Claude decide when to invoke it.
  • Adaptive thinking lets Claude reason internally before answering. Quality lift is real on hard tasks; latency cost is also real.
  • Batch processing is 50% off the standard rate for non-real-time workloads. Combined with prompt caching you can stack savings to 90%.
  • The model family matters more than people realize. Opus 4.7 for the hardest work. Sonnet 4.6 for most production traffic. Haiku 4.5 for fast and cheap.

Most asks don’t need Opus. Sonnet 4.6 is enough for the majority of work and runs at a fraction of the price. The skill of being a good Claude API user is, partly, knowing which model to send each request to.

Cowork is the newest surface and the most ambitious

Cowork is the desktop product that gives Claude direct file system access. Inside a sandboxed VM, Claude can read your documents, edit them in place, and create new ones. Stop pasting context. Claude opens the file.

The four features that matter:

  • Local file access: the foundation. Claude reads and writes real files on your machine.
  • Computer Use: Claude operates your mouse, browser, and apps. Slow but unblocks the long tail of apps without APIs.
  • Scheduled tasks: Claude on a cron. Every Friday, organize Downloads. Every weekday at 8am, prep meeting briefs.
  • Dispatch: bridge between mobile and desktop. Send instructions from your phone, the desktop session picks them up.

Cowork is where the line between “I asked Claude to” and “Claude already did” gets thin.

Foundations are the physics

Some things are not in any single drawer. They show up everywhere.

  • The 1 million token context window is generally available now. Whole codebases in working memory.
  • MCP is the same protocol behind every connector, every tool, every integration. Learn it once, understand the whole family.
  • Adaptive thinking is everywhere modern Claude lives. Same primitive, different surfaces.
  • The model family is the same brain at three speed-and-cost points.

If you understand the foundations, the surfaces stop feeling like separate products and start feeling like different doors into the same building.

How I actually use all of this

Most days I’m in claude.ai working in a Project. Memory and Preferences are doing quiet work in the background. When I need to write code, I switch to Claude Code with my CLAUDE.md and a few Skills set up. When I’m building something for production, I’m calling the API with Sonnet 4.6 and tool use. Cowork is newer; I’m using it for scheduled file cleanup and morning meeting briefs.

The pattern: pick the surface that matches the friction. Chat for thinking. Code for shipping. API for embedding. Cowork for desktop chores.

You don’t need to learn it all on day one. Start with claude.ai Projects. Add what you need when the friction shows up.
